# Lumeo — Платформа интерактивного видеообучения

> Дипломная работа: «Разработка отечественной веб-платформы интерактивного видеообучения»

Lumeo — это корпоративная LMS-платформа для размещения обучающих видео с встроенными интерактивными вопросами, автоматической генерацией субтитров на основе ИИ и детальной аналитикой прогресса студентов.

## Возможности

### Для студентов
- Просмотр видео с интерактивными вопросами прямо во время воспроизведения
- Автосубтитры, сгенерированные на основе ИИ (Whisper)
- Прохождение курсовых тестов с ограничением попыток и проходным баллом
- Отслеживание прогресса и история просмотров
- Запись на курсы (открытая / по заявке)

### Для преподавателей
- Создание курсов и загрузка видео (MP4, WebM, MKV, MOV, AVI, WMV, OGV)
- Редактор интерактивных событий: вопросы с единственным / множественным выбором и свободным ответом
- Автоматическая генерация субтитров (`.vtt`) + ручное редактирование
- Курсовые тесты с drag-and-drop упорядочиванием вопросов
- Аналитика по каждому студенту и каждому вопросу, экспорт в Excel
- Управление соавторами курса и заявками на зачисление (real-time уведомления)

### Для администраторов
- Управление пользователями: создание, редактирование, удаление, импорт/экспорт Excel
- Настройка LDAP/Active Directory, OAuth 2.0 (Яндекс, Google), SAML SSO
- Брендинг платформы: логотип, цвета, фоновые паттерны
- Мониторинг сервера: CPU, RAM, диск, аптайм, системные логи
- Очистка кэша ИИ-модели, резервное копирование БД

## Стек технологий

| Слой | Технологии |
|---|---|
| **Frontend** | React 19, TypeScript 5.9, Vite 7, React Router v7 |
| **Backend** | Node.js 20, Express v5, TypeScript |
| **База данных** | PostgreSQL 15, Sequelize-typescript (ORM) |
| **Аутентификация** | JWT, Passport.js — Local / LDAP / OAuth 2.0 / SAML |
| **ИИ / ML** | Xenova/transformers (Whisper-small — субтитры, MiniLM — оценка ответов) |
| **Real-time** | Server-Sent Events (SSE) |
| **Медиа** | FFmpeg, Multer (до 500 МБ) |
| **Деплой** | Docker, Docker Compose, nginx |

## Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                        Браузер                          │
│              React SPA (Vite → nginx:80)                │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP / SSE
                    nginx reverse proxy
                    /api  →  backend:5001
                    /uploads → static files
                         │
┌────────────────────────▼────────────────────────────────┐
│               Node.js / Express (порт 5001)             │
│                                                         │
│  ┌──────────┐  ┌────────────┐  ┌──────────────────────┐ │
│  │  Routes  │→ │Controllers │→ │  Sequelize Models    │ │
│  └──────────┘  └────────────┘  └──────────────────────┘ │
│                      │                    │              │
│  ┌───────────────┐   │          ┌─────────▼──────────┐  │
│  │ subtitleWorker│←──┘          │   PostgreSQL 15     │  │
│  │ (Worker thread│              └────────────────────┘  │
│  │  Whisper AI)  │                                       │
│  └───────────────┘                                       │
└─────────────────────────────────────────────────────────┘
```

**Поток аутентификации:**
```
Логин → JWT-токен → Authorization: Bearer <token> → checkAuth middleware → req.user
```

**Потоки real-time (SSE):**
| Канал | Назначение |
|---|---|
| `GET /api/videos/:videoId/events/stream` | Интерактивные события видео |
| `GET /api/videos/enrollment/stream` | Статус заявки студента |
| `GET /api/videos/courses/:courseId/enrollment/stream` | Новые заявки (для преподавателя) |
| `GET /api/videos/courses/:courseId/processing/stream` | Готовность субтитров |
| `GET /api/users/admin/stream` | Новые заявки пользователей (для admin) |

## Быстрый старт (Docker)

### Требования
- Docker 24+
- Docker Compose v2

### 1. Клонировать репозиторий
```bash
git clone <repo-url> && cd VKR
```

### 2. Настроить окружение
```bash
cp .env.example .env
# Отредактировать .env: задать DB_PASSWORD, JWT_SECRET, ADMIN_PASSWORD
```

Сгенерировать JWT_SECRET:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 3. Запустить (production)
```bash
docker compose up -d --build
```

Платформа будет доступна по адресу: **http://localhost**

Учётная запись администратора по умолчанию:
- Email: `admin@lumeo.local`
- Пароль: значение `ADMIN_PASSWORD` из `.env`

> **Если `ADMIN_PASSWORD` не задан** — при первом запуске сервер автоматически сгенерирует криптографически случайный пароль и **выведет его в лог**.
>
> Посмотреть пароль:
> ```bash
> docker compose logs server | grep "Пароль"
> # или, если запускаете напрямую:
> npm run dev 2>&1 | grep "Пароль"
> ```
>
> Пример вывода в логах:
> ```
> ⚠️  ADMIN_PASSWORD не задан в .env — сгенерирован случайный пароль.
>     Логин:  admin@lumeo.local
>     Пароль: a3f8c2d1e9b74051...
>     Сохраните его и смените через профиль!
> ```
>
> После входа **обязательно** смените пароль в разделе «Профиль».

### 4. Запустить (разработка с hot-reload)
```bash
docker compose -f docker-compose.dev.yml up --build
```

| Сервис | Адрес |
|---|---|
| Frontend (Vite HMR) | http://localhost:5173 |
| Backend API | http://localhost:5001 |

## Конфигурация окружения

Все переменные описаны в `.env.example`. Обязательные:

| Переменная | Описание |
|---|---|
| `DB_NAME` / `DB_USER` / `DB_PASSWORD` | Параметры PostgreSQL |
| `DB_HOST` | `postgres` (Docker) или `localhost` (локально) |
| `JWT_SECRET` | Секрет подписи JWT, минимум 32 символа |
| `API_URL` | Публичный URL сервера (для формирования ссылок на файлы) |
| `CLIENT_URL` | URL клиента (CORS + OAuth-редиректы) |
| `ADMIN_PASSWORD` | Пароль первого admin при инициализации |

Опциональные (OAuth):

| Переменная | Описание |
|---|---|
| `YANDEX_CLIENT_ID` / `YANDEX_CLIENT_SECRET` | OAuth через Яндекс ID |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | OAuth через Google |

LDAP и SAML настраиваются динамически через Admin Panel → Системные настройки.
